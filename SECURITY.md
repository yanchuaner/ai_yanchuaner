# 安全策略

## 披露安全问题

请不要为未修复漏洞创建公开 Issue，也不要在聊天、截图或日志中粘贴真实 API Key、OIDC Secret、Cookie、数据库备份或用户对话。请通过仓库 Security Advisory 私密报告，并提供受影响版本、复现条件、影响范围和脱敏证据。

维护者应确认收到、完成分级、制定修复与轮换方案，并在修复部署后协调披露。发现凭据泄露时，先撤销或轮换凭据，再清理历史和日志；仅删除 Git 当前文件不足以消除泄露。

## 生产红线

- 主站是普通用户唯一身份源；Open WebUI 本地注册和匿名访问保持关闭。
- LiteLLM 管理界面、PostgreSQL 和内部 Docker 网络不得暴露公网。
- 生产密钥只通过环境变量或 Secret 注入，不进入仓库、镜像、日志或备份清单。
- 上游密钥与未来 BYOK 必须强加密保存；管理员日常只能查看脱敏信息和健康状态。
- Open WebUI 会保存账号、对话、文件与知识库数据，必须在上线前明确保留期、用户删除能力和备份访问边界。
- Open WebUI 的燕中品牌配置在未取得书面或企业许可前，只能用于滚动 30 日直接用户不超过 50 人的部署。

## 发布检查

运行 Compose 配置解析、健康检查、受限 Key 冒烟测试、备份恢复演练和敏感信息扫描；核对镜像 digest 与 `THIRD_PARTY_NOTICES.md` 一致。任何一项无法验证时，不得把预览环境描述为已通过生产验收。
